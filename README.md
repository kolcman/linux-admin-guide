# Инструменты для DevOps

Офлайн-шпаргалка в терминале для тех, кто начинает с нуля или работает на удалённом сервере по SSH.  
Один бинарник — без браузера, без GUI, без лишних зависимостей.

Скачал → запустил → нашёл команду с пояснением.

---

## Зачем это нужно

На сервере часто нет удобной документации под рукой. Поиск в интернете отвлекает, man-страницы пугают новичков.  
Этот инструмент даёт **структурированные подсказки**: что делает команда, какие флаги важны, на что смотреть в первую очередь.

Подходит для:

- junior-админов и DevOps-новичков  
- быстрого напоминания на продакшен-сервере  
- обучения прямо в SSH-сессии

## Быстрый старт

```bash
# Скачать
wget https://github.com/kolcman/linux-admin-guide/releases/latest/download/linux-guide

# Запустить
chmod +x linux-guide
./linux-guide
```

### Установка в систему

```bash
sudo mv linux-guide /usr/local/bin/
linux-guide
```

Или через скрипт:

```bash
curl -fsSL https://raw.githubusercontent.com/kolcman/linux-admin-guide/main/scripts/install.sh | bash
```

### Обновление

```bash
wget -O linux-guide https://github.com/kolcman/linux-admin-guide/releases/latest/download/linux-guide
chmod +x linux-guide
sudo mv linux-guide /usr/local/bin/
```

## Управление

| Клавиша | Действие |
|---------|----------|
| `↑` `↓` или `j` `k` | Навигация по меню / скролл |
| `Enter` | Открыть раздел |
| `Esc` | Назад |
| `q` или `й` | Выход / назад (работает и на русской раскладке) |

В разделах команды подсвечиваются: **флаги** — жёлтым, **аргументы** — голубым, пояснения — под строкой.

## Сборка из исходников

Требуется Go 1.22+.

```bash
git clone https://github.com/kolcman/linux-admin-guide.git
cd linux-admin-guide
go build -o linux-guide .
./linux-guide
```

## Требования

| | |
|---|---|
| ОС | Linux (x86_64 / amd64) |
| Терминал | SSH или локальный |
| Зависимости | нет (статический бинарник) |
| Интернет | только для установки и обновления |

После установки работает **полностью офлайн**.
